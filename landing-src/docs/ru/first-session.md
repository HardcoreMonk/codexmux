purplemux уже запущен (если нет — см. [Быстрый старт](/purplemux/ru/docs/quickstart/)). Эта страница рассказывает, что именно делает интерфейс, чтобы первые несколько минут были менее абстрактными.

## Панель управления

Когда вы открываете `http://localhost:8022`, попадаете в **рабочее пространство**. Считайте его папкой связанных вкладок — одно для проекта, который вы пишете с Claude, другое для документации, третье для разовой шелл-работы.

Раскладка:

- **Боковая панель слева** — рабочие пространства и сессии, бейджи статуса Claude, виджет лимитов, заметки, статистика
- **Центральная область** — панели текущего рабочего пространства; в каждой панели может быть несколько вкладок
- **Верхняя строка** — имя рабочего пространства, элементы управления разделением, настройки

Боковую панель можно свернуть в любой момент через <kbd>⌘B</kbd>. Переключение режима Workspace/Sessions внутри неё — <kbd>⌘⇧B</kbd>.

## Создайте рабочее пространство

При первом запуске у вас уже есть одно рабочее пространство по умолчанию. Чтобы добавить ещё:

1. Нажмите **+ Новое рабочее пространство** в верхней части боковой панели (<kbd>⌘N</kbd>).
2. Назовите его и выберите каталог по умолчанию — оттуда будут стартовать шеллы новых вкладок.
3. Нажмите Enter. Откроется пустое рабочее пространство.

Позже рабочие пространства можно переупорядочивать и переименовывать, перетаскивая в боковой панели.

## Откройте первую вкладку

Свежее рабочее пространство пустое. Добавьте вкладку через <kbd>⌘T</kbd> или кнопку **+** на панели вкладок.

Выберите **шаблон**:

- **Terminal** — пустая оболочка. Подходит для `vim`, `docker`, скриптов.
- **Claude** — стартует с уже запущенным `claude`.

{% call callout('tip', 'Шаблоны — это просто ярлыки') %}
Под капотом любая вкладка — это обычная оболочка. Шаблон Claude — это просто «открыть терминал и запустить `claude`». Если вы позже сами запустите `claude` во вкладке Terminal, purplemux это заметит и точно так же начнёт показывать его статус.
{% endcall %}

## Считайте статус сессии

Посмотрите на **строку сессии в боковой панели** для своей вкладки. Вы увидите один из индикаторов:

| Состояние | Что значит |
|---|---|
| **Idle** (серый) | Claude ждёт вашего ввода. |
| **Busy** (фиолетовый спиннер) | Claude работает — читает файлы, запускает инструменты. |
| **Needs input** (янтарный) | Claude упёрся в запрос разрешения или задал вопрос. |
| **Review** (синий) | Работа завершена, Claude остановился; есть что проверить. |

Переходы практически мгновенные. Подробнее о том, как это определяется, — в [Статусе сессии](/purplemux/ru/docs/session-status/).

## Ответьте на запрос разрешения

Когда Claude просит запустить инструмент или отредактировать файл, purplemux **перехватывает запрос** и показывает его прямо в виде сессии. Вы можете:

- Кликнуть **1 · Да** / **2 · Да, всегда** / **3 · Нет**, или
- Нажать соответствующую цифру на клавиатуре, или
- Проигнорировать и ответить с телефона — мобильный Web Push пришлёт ту же подсказку.

Сам Claude CLI на перехваченном запросе не блокируется на самом деле; purplemux передаёт ему ваш ответ.

## Разделение и переключение

Когда вкладка работает, попробуйте:

- <kbd>⌘D</kbd> — разделить текущую панель вправо
- <kbd>⌘⇧D</kbd> — разделить вниз
- <kbd>⌘⌥←/→/↑/↓</kbd> — переключить фокус между разделениями
- <kbd>⌘⇧[</kbd> / <kbd>⌘⇧]</kbd> — предыдущая / следующая вкладка

Полный список — на странице [Клавиатурные сокращения](/purplemux/ru/docs/keyboard-shortcuts/).

## Сохранение и восстановление

Закройте браузер. Ваши вкладки никуда не исчезнут — tmux держит их открытыми на сервере. Зайдите через час (или через неделю), и purplemux восстановит точно ту же раскладку, включая пропорции разделений и рабочие каталоги.

Даже перезагрузка сервера восстанавливается: при старте purplemux читает сохранённую раскладку из `~/.purplemux/workspaces.json`, перезапускает шеллы в нужных каталогах и, где возможно, переподключает сессии Claude.

## Доступ со смартфона

Запустите:

```bash
tailscale serve --bg 8022
```

На смартфоне откройте `https://<machine>.<tailnet>.ts.net`, нажмите **Поделиться → На экран «Домой»** и разрешите уведомления. Теперь вы получаете push-сигналы для состояний **needs-input** и **review**, даже когда вкладка закрыта.

Полное руководство: [Настройка PWA](/purplemux/ru/docs/pwa-setup/) · [Web Push](/purplemux/ru/docs/web-push/) · [Tailscale](/purplemux/ru/docs/tailscale/).
