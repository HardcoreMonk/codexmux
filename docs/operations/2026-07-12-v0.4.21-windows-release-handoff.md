## 범위

- 실제 `v0.4.20` installer를 baseline으로 fresh Windows package와 local updater를
  검증했습니다.
- 검증한 네 자산을 prerelease로 게시하고 exact target tag `v0.4.21`의 published
  channel과 실제 `quitAndInstall`을 통과한 뒤 stable/latest로 승격했습니다.
- Browser reconnect, Windows package와 published updater evidence는 upload 전에 별도
  privacy scanner를 통과했습니다.
- Workflow 진행 중과 직후에 세 artifact 다운로드본을 확보하고, stable 승격 후 총 16개
  JSON을 함께 독립 재검사했습니다.

## Upload integrity 증거

Fresh Windows package gate에서 다음 항목이 모두 통과했습니다.

- Exact-limit artifact의 byte size와 SHA-256이 request body와 일치했습니다.
- Stage와 final이 같은 directory에 있었고 no-replace hard-link publish가 성공했습니다.
- Reserved stage를 실행 중 관찰한 뒤 abort했고 동일 path가 unlink됐습니다.
- Age floor를 넘긴 reserved stage는 cleanup됐습니다.
- 정상 committed `*.part` artifact는 reserved-stage cleanup 뒤에도 유지됐습니다.
- `CODEXMUX_UPLOADS_DISABLED=1` instance는 exact 두 upload route만 `503`을 반환하고 health,
  auth와 기존 artifact를 보존했습니다.
- Runtime output과 browser console blocker는 없었습니다.

## Artifact privacy 증거

Workflow는 browser, Windows package와 published updater job에서 JSON upload 직전에
`check:smoke-artifacts`를 실행했습니다. 금지 key, URL, Codex session path, Windows/Linux
smoke temp path 또는 terminal escape가 있으면 해당 artifact upload와 stable promotion을
중단하는 fail-closed 구조입니다.

Workflow 진행 중과 직후에 확보한 세 evidence artifact 다운로드본의 총 16개 JSON을 stable
승격 후 같은 scanner로 함께 독립 검사했고 모두 통과했습니다. Published updater artifact의
세 JSON도 별도로 검사했으며 raw URL, child stdout/stderr tail, 임시 path와 terminal escape를
찾지 못했습니다. Scanner 출력은 raw value가 아니라 basename, JSON path와 위반 label만
사용합니다.

## v0.4.20 교정과의 관계

`v0.4.20`의 package/upload와 실제 updater 적용 기능 결과는 유효합니다. 다만 release 후
재감사에서 `smoke-windows-published-update-v0.4.20`의 JSON 2개가 URL, temp path와 terminal
escape 때문에 privacy scanner에 실패했습니다. Token이나 credential은 발견되지 않았습니다.

Commit `b4b1af13`은 child tail/raw message/URL/path를 sanitizer에서 제거하고 세 release job에
upload 전 scanner를 추가했습니다. 이 `v0.4.21` handoff가 published-updater privacy-safe
evidence의 현재 기준이며, `v0.4.20` handoff는 최초 기능 검증과 교정 기록으로 유지합니다.

## Rollback과 제한

- 이 release는 unsigned 내부 배포 build입니다. Public code-signing trust와 SmartScreen
  reputation은 검증 범위가 아니며 내부 신뢰 절차와 installer 경고가 필요합니다.
- macOS package와 npm publish는 Windows stable gate에서 제외했습니다. 이 handoff는 해당
  surface의 배포 성공을 의미하지 않습니다.
- Upload 장애 시 `CODEXMUX_UPLOADS_DISABLED=1`로 재시작해 두 upload route만 `503`으로
  격리하고 patched version으로 forward recovery합니다.
- Updater 자동 downgrade와 `v0.4.21 -> v0.4.20` 호환성은 검증하지 않았습니다. 문제가 있으면
  영향 surface를 격리한 뒤 새 version으로 forward fix하고 전체 gate를 반복합니다.
- 다음 stable release는 workflow baseline tag, version과 installer SHA-256을 `v0.4.21`로
  다시 pin해야 합니다.
