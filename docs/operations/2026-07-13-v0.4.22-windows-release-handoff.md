# v0.4.22 Windows release handoff

날짜: 2026-07-13
상태: stable/latest 게시, Windows update와 privacy-safe evidence 검증 및 운영 인계 완료
관련 결정: `ADR-027`, `ADR-028` Verified; `ADR-029` Implemented

## 범위

- 실제 `v0.4.21` installer를 baseline으로 fresh Windows package와 local updater를
  검증했습니다.
- 검증한 네 자산을 prerelease로 게시하고 exact target tag `v0.4.22`의 published
  channel과 실제 `quitAndInstall`을 통과한 뒤 stable/latest로 승격했습니다.
- Browser reconnect, Windows package와 published updater evidence는 upload 전에 privacy
  scanner를 통과했고, 세 artifact의 16개 JSON을 함께 독립 재검사했습니다.
- `codexmux-session-token` namespace와 같은 hostname의 legacy cookie 공존 회귀를 release에
  포함했습니다.

## Cookie namespace 증거와 경계

Chromium reconnect smoke는 Codexmux cookie를 `codexmux-session-token`으로 설정한 뒤 같은
hostname에 legacy `session-token`을 추가하고 page 인증과 Runtime v2 terminal WebSocket
recovery가 유지되는지 확인했습니다. Fresh Windows package와 published updater도 새 cookie
contract가 포함된 `v0.4.22` 자산과 post-update health를 검증했습니다.

이 증거는 isolated/fresh HOME과 browser profile을 사용합니다. 따라서 다음 동작은 아직
실제 전환 증거가 아닙니다.

- 기존 `v0.4.21` Electron profile이 update 직후 login으로 수렴하는지
- Codexmux 1회 로그인 뒤 기존 workspace와 Runtime v2 session에 다시 attach되는지
- Legacy cookie가 이전 Codexmux JWT일 때 Purplemux 1회 로그인으로 양쪽 인증이 유지되는지
- 기존 profile에서 session-authenticated upload와 logout이 Codexmux cookie만 사용하는지

Release note의 1회 재로그인 안내는 설계상 필요한 운영 지침이며, 위 old-profile 동작을
관찰했다는 뜻이 아닙니다. 해당 evidence를 확보할 때까지 ADR-029는 `Implemented` 상태를
유지합니다.

## Artifact privacy 증거

Workflow는 browser, Windows package와 published updater job에서 JSON upload 직전에
`check:smoke-artifacts`를 실행했습니다. 금지 key, URL, Codex session path, Windows/Linux
smoke temp path 또는 terminal escape가 있으면 artifact upload와 stable promotion을
중단하는 fail-closed 구조입니다.

세 evidence artifact 다운로드본의 총 16개 JSON을 stable 승격 뒤 같은 scanner로 독립
검사했고 모두 통과했습니다. Scanner 출력은 raw value가 아니라 basename, JSON path와 위반
label만 사용합니다.

## Rollback과 제한

- 이 release는 unsigned 내부 배포 build입니다. Public code-signing trust와 SmartScreen
  reputation은 검증 범위가 아니며 내부 신뢰 절차와 installer 경고가 필요합니다.
- macOS package와 npm publish는 Windows stable gate에서 제외했습니다.
- Upload 장애 시 `CODEXMUX_UPLOADS_DISABLED=1`로 재시작해 두 upload route만 `503`으로
  격리하고 patched version으로 forward recovery합니다.
- 구버전으로 downgrade하면 generic cookie 충돌이 다시 발생할 수 있습니다. 긴급 rollback 중
  두 앱을 함께 사용해야 하면 서로 다른 hostname으로 접속하고 각 앱에 다시 로그인합니다.
- Post-release `main` workflow의 다음 baseline tag/version은 `v0.4.22`로, installer
  SHA-256은 `84d0480b227113a776c5cd92f94ebe007171aef474be138e88f095ce0e0cdd35`로 pin했습니다.
