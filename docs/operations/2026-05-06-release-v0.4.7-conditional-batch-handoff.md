The first Electron runtime v2 run failed because the smoke's isolated server used the dev `tsx
server.ts` path while a built standalone tree was present. The smoke now auto-selects
`bin/codexmux.js` production launch when `.next/standalone/server.js` exists, with
`CODEXMUX_ELECTRON_RUNTIME_V2_SERVER_MODE=production|dev` as an override.

Real iPad Home Screen long-background, input draft preservation, and macOS packaged
Finder/Gatekeeper UX still require their actual devices/sessions. This Linux host cannot produce
those manual UX artifacts.
