# Phase 4 — Pane (분할) PRD

## 목표

화면 분할로 여러 터미널을 동시에 표시하는 것.

Phase 3에서 구현한 탭 기반 단일 Pane을 확장하여, 화면을 수평/수직으로 분할하고 각 Pane이 독립적인 Surface(탭) 그룹을 가지도록 한다. 사용자는 여러 터미널을 동시에 보며 작업할 수 있다.

## 완료 조건

화면을 분할하여 여러 터미널을 동시에 보며 작업할 수 있다. 각 Pane은 독립적인 탭 그룹을 가지며, Pane 간 포커스 이동이 가능하다.

---

## 현재 상태 (Phase 3 완료)

```
┌─ Tab 1 ──┬─ Tab 2 ──┬─ Tab 3 ──┬─ + ─┐
│                                        │
│          활성 탭의 터미널                 │
│          (단일 Pane, 단일 xterm.js)     │
│                                        │
└────────────────────────────────────────┘
```

- 단일 Pane에 여러 탭(Surface)
- 탭 전환 시 WebSocket 재연결 + 단일 xterm.js 재활용
- `~/.purple-terminal/tabs.json`에 탭 상태 저장
- 활성 탭의 터미널만 화면에 표시

## 목표 상태 (Phase 4)

```
┌─ Tab A1 ─┬─ Tab A2 ─┬─ + ─┐│┌─ Tab B1 ─┬─ + ─┐
│                             ││                   │
│     Pane A (활성 탭 터미널)   ││  Pane B (터미널)   │
│     (포커스)                 ││                   │
│                             ││                   │
├─────────────────────────────┤│                   │
│┌─ Tab C1 ─┬─ Tab C2 ─┬─ + ┐││                   │
│                             ││                   │
│     Pane C (터미널)          ││                   │
│                             ││                   │
└─────────────────────────────┘└───────────────────┘
```

- 여러 Pane을 수평/수직으로 분할
- 각 Pane에 독립적인 탭 그룹
- 각 Pane에 독립적인 xterm.js 인스턴스
- Pane 간 드래그 리사이즈
- 포커스된 Pane의 터미널만 키 입력을 수신

## 요구사항

### REQ-1: Pane 레이아웃 구조

Pane 분할 상태를 트리 구조로 관리한다.

- 레이아웃을 이진 트리(binary tree)로 표현한다
  - **리프 노드**: 개별 Pane (탭 그룹 + 터미널)
  - **내부 노드**: 분할 컨테이너 (방향 + 비율 정보)
- 분할 컨테이너는 방향(horizontal/vertical)과 분할 비율을 가진다
- 초기 상태는 단일 Pane (루트 = 리프 노드)
- **최대 Pane 수: 3개** — 리프 노드가 3개에 도달하면 추가 분할을 비활성화한다
- **최소 Pane 크기**: 너비 200px, 높이 120px
- Phase 3의 단일 Pane 레이아웃과 하위 호환: Pane이 1개인 경우 Phase 3과 동일한 UX

### REQ-2: Pane 분할

현재 Pane을 수평 또는 수직으로 분할하여 새 Pane을 생성한다.

- 수직 분할: 현재 Pane을 좌/우로 나눈다
- 수평 분할: 현재 Pane을 상/하로 나눈다
- 분할 시 현재 Pane이 내부 노드로 변환되고, 기존 내용은 첫 번째 자식 Pane으로, 새 Pane은 두 번째 자식 Pane으로 생성된다
- 새 Pane은 기본 탭 1개("Terminal 1")와 새 tmux 세션으로 시작한다
- 새 Pane의 터미널은 원래 Pane의 현재 작업 디렉토리를 유지한다
- 분할 시 초기 비율은 50:50으로 설정한다
- 분할은 UI 버튼으로 실행한다
- Pane이 이미 3개이면 분할 버튼을 비활성화한다

### REQ-3: Pane 리사이즈

`react-resizable-panels` 라이브러리를 사용하여 분할선 드래그로 Pane 크기를 조절한다.

- `PanelGroup` + `Panel` + `PanelResizeHandle` 컴포넌트 기반 구현
- `PanelResizeHandle`이 히트 영역 및 드래그 인터랙션을 자동 처리 (마우스, 터치, 키보드)
- `Panel`의 `minSize` prop으로 최소 비율 10% 하한 설정
- 리사이즈 시 해당 Pane의 xterm.js와 tmux 세션 크기를 동기화한다 (resize 메시지 전송)
- `PanelGroup`의 `onLayout` 콜백으로 비율 변경 감지 → `layout.json`에 저장 (디바운스)

### REQ-4: Pane 닫기

Pane을 닫으면 해당 Pane의 모든 탭과 tmux 세션이 종료된다.

- Pane 닫기 시 해당 Pane의 모든 tmux 세션을 `kill-session`으로 종료한다
- 닫힌 Pane의 형제 Pane이 부모 노드의 영역 전체를 차지한다 (트리 재구성)
- 마지막 Pane은 닫을 수 없다 (빈 화면 방지)
- Pane 닫기는 UI 버튼으로 실행한다

### REQ-5: Pane 포커스 관리

포커스된 Pane만 키 입력을 수신한다.

- Pane 클릭 시 해당 Pane에 포커스를 설정한다
- 포커스된 Pane은 시각적으로 구분한다 (보더 하이라이트 등)
- 단축키로 포커스를 인접 Pane으로 이동할 수 있다 (상/하/좌/우)
- 포커스된 Pane의 활성 탭 터미널만 키 입력을 수신한다
- 새 Pane 생성 시 새 Pane에 자동으로 포커스를 설정한다

### REQ-6: Pane별 독립 xterm.js 인스턴스

각 Pane이 독립적인 xterm.js 인스턴스를 가진다.

- Phase 3의 단일 xterm.js 재활용 방식에서, Pane당 1개의 xterm.js 인스턴스로 전환한다
- 각 Pane은 활성 탭의 WebSocket 연결을 독립적으로 관리한다
- 탭 전환은 Phase 3과 동일하게 해당 Pane의 WebSocket을 끊고 재연결한다
- Pane 생성 시 xterm.js 인스턴스를 생성하고, Pane 닫기 시 정리한다
- 각 xterm.js 인스턴스는 Pane 크기에 맞게 cols/rows를 계산한다

### REQ-7: Pane별 독립 탭 그룹

각 Pane이 독립적인 Surface(탭) 그룹을 가진다.

- 각 Pane은 자체 탭 바, 탭 목록, 활성 탭을 관리한다
- 탭 생성/전환/삭제/순서 변경/이름 변경은 Phase 3과 동일한 동작을 Pane 단위로 수행한다
- Pane의 마지막 탭에서 `exit` 실행 시: Pane이 1개면 새 탭 자동 생성, 복수 Pane이면 해당 Pane 닫기
- 각 Pane의 탭 상태는 레이아웃과 함께 저장한다

### REQ-8: 레이아웃 영속성

Pane 분할 상태를 파일에 저장하여 서버 재시작/새로고침 후 복원한다.

- Phase 3의 `tabs.json` 단순 배열 구조를 Pane 트리 구조로 확장한다
- `~/.purple-terminal/layout.json`에 전체 레이아웃 트리를 저장한다
  - 트리 구조 (분할 방향, 비율)
  - 각 Pane의 탭 목록 (탭 ID, 세션 이름, 탭 이름, 순서)
  - 각 Pane의 활성 탭
  - 포커스된 Pane ID
- 레이아웃 변경(분할, 닫기, 리사이즈, 탭 변경) 시 자동 저장 (디바운스 적용)
- 서버 시작 시 `layout.json` + tmux 세션 크로스 체크하여 복원
  - layout.json에 있지만 tmux 세션이 없는 탭 → 제거
  - tmux 세션이 있지만 layout.json에 없는 세션 → 기본 Pane에 탭으로 추가 (orphan 복구)
- Phase 3의 `tabs.json`이 존재하고 `layout.json`이 없는 경우: `tabs.json`을 단일 Pane 레이아웃으로 마이그레이션

### REQ-9: 작업 디렉토리 유지

새 Pane 생성 시 원래 Pane의 작업 디렉토리를 유지한다.

- 분할 시 원래 Pane의 활성 tmux 세션에서 현재 작업 디렉토리(CWD)를 조회한다
- 새 tmux 세션 생성 시 해당 디렉토리에서 쉘을 시작한다
- CWD 조회: `tmux display-message -p -t {session} '#{pane_current_path}'`

## 비기능 요구사항

### NFR-1: 분할/리사이즈 성능

Pane 분할과 리사이즈가 즉각적으로 느껴져야 한다. 리사이즈 중 터미널 렌더링이 끊기지 않아야 한다.

### NFR-2: 다중 Pane 메모리

Pane당 xterm.js 인스턴스를 유지하므로 메모리 사용량이 증가한다. 합리적인 Pane 수(4~6개)에서 문제가 없어야 한다.

### NFR-3: 독립 세션 유지

각 Pane의 비활성 탭 tmux 세션이 백그라운드에서 계속 실행되어야 한다. Pane 간 세션이 간섭하지 않아야 한다.

### NFR-4: Phase 3 호환

단일 Pane 상태에서의 UX가 Phase 3과 동일해야 한다. 탭 관리 동작(생성/전환/삭제/순서 변경/이름 변경)이 변경되지 않아야 한다.

### NFR-5: 레이아웃 정합성

서버 재시작/새로고침 후 Pane 분할 구조, 각 Pane의 탭 목록, 포커스 상태가 모두 정확하게 복원되어야 한다.

### NFR-6: 포커스 표시

포커스된 Pane이 시각적으로 명확하게 구분되어야 한다. Muted 팔레트에 맞는 보더 하이라이트를 적용한다.

## 검증 시나리오

1. **수직 분할**: 분할 버튼 클릭 시 화면이 좌/우로 나뉘고 각각 독립 터미널이 동작한다
2. **수평 분할**: 분할 버튼 클릭 시 화면이 상/하로 나뉘고 각각 독립 터미널이 동작한다
3. **중첩 분할**: Pane A를 수직 분할 → 생성된 Pane B를 수평 분할 → 3개 Pane이 정상 동작한다
4. **Pane별 탭 관리**: 각 Pane에서 독립적으로 탭 생성/전환/삭제가 가능하다
5. **Pane 리사이즈**: 분할선 드래그 시 Pane 크기가 변경되고 터미널이 리사이즈된다
6. **Pane 닫기**: Pane 닫기 시 해당 Pane의 모든 세션이 종료되고 나머지 Pane이 영역을 채운다
7. **마지막 Pane 보호**: 마지막 남은 Pane은 닫을 수 없다
8. **포커스 이동**: Pane 클릭 또는 단축키로 포커스가 이동하고 키 입력은 포커스된 Pane에만 전달된다
9. **작업 디렉토리 유지**: 특정 디렉토리에서 분할 시 새 Pane의 터미널이 같은 디렉토리에서 시작된다
10. **서버 재시작 복원**: 서버 재시작 후 Pane 분할 구조, 각 Pane의 탭, 포커스 상태가 모두 복원된다
11. **새로고침 복원**: 브라우저 새로고침 후 레이아웃이 그대로 복원된다
12. **다중 Pane 동시 실행**: 여러 Pane에서 각각 프로세스 실행 중 모두 독립적으로 동작한다
13. **단일 Pane 호환**: Pane이 1개인 상태에서 Phase 3과 동일한 UX가 제공된다
14. **tabs.json 마이그레이션**: Phase 3에서 업그레이드 시 기존 탭이 단일 Pane 레이아웃으로 정상 복원된다
15. **Pane 내 exit**: 복수 Pane에서 마지막 탭에 exit 실행 시 해당 Pane이 닫힌다

## 확정된 결정사항

| 항목 | 결정 | 근거 |
|---|---|---|
| 레이아웃 구조 | 이진 트리 (리프=Pane, 내부=분할 컨테이너) | 수평/수직 중첩 분할을 자연스럽게 표현 |
| xterm.js 전략 | Pane당 독립 인스턴스 | 동시 표시를 위해 필수, 단일 재활용은 불가 |
| WebSocket 전략 | Pane당 독립 연결 (활성 탭) | 각 Pane이 독립적으로 터미널 표시 |
| 상태 저장 | `layout.json` (트리 구조) | 플랫 배열로는 분할 구조 표현 불가 |
| tabs.json 마이그레이션 | layout.json 없을 시 자동 변환 | Phase 3 → Phase 4 무중단 업그레이드 |
| 분할 초기 비율 | 50:50 | 가장 자연스러운 기본값 |
| 새 Pane CWD | 원래 Pane의 활성 세션 CWD 유지 | 같은 프로젝트 디렉토리에서 작업 연속성 |
| 최소 Pane 비율 | 10% | 너무 작은 Pane 방지 |
