# Phase 10 — Web 입력창 PRD

## 목표

타임라인과 터미널 사이에 웹 입력창을 두어 Claude Code에 편하게 입력할 수 있게 하는 것.

현재(Phase 9까지)는 Claude Code에 입력하려면 하단 축소 터미널에 직접 타이핑해야 한다. 축소된 터미널은 타이핑하기 불편하고, 여러 줄 입력이 어렵다. Phase 10에서는 타임라인과 터미널 사이에 전용 웹 입력창을 배치하여, 일반 채팅 앱처럼 편하게 입력하고 전송할 수 있다.

## 완료 조건

Claude Code Panel 모드에서 웹 입력창에 타이핑하면 하단 터미널에 그대로 입력되어 실행.

---

## 현재 상태 (Phase 9 완료)

### 이미 구현된 것

- Claude Code Panel: 타임라인 뷰 + 축소 터미널
- 세션 목록 탐색 + resume 연결
- 세션 ID 영속화 + 서버 재시작 자동 resume
- 타임라인 실시간 업데이트 (WebSocket + fs.watch)
- Panel 타입 수동/자동 전환

### Phase 10에서 추가할 것

- 타임라인과 터미널 사이에 텍스트 입력 영역 배치
- Enter로 전송 → 하단 터미널(pty)에 텍스트 전달
- 여러 줄 입력 지원 (Shift+Enter로 줄바꿈)
- 입력창 포커스 단축키
- Claude Code 비활성 시 입력창 숨김

---

## 요구사항

### REQ-1: 입력창 배치

타임라인과 터미널 사이에 텍스트 입력 영역을 배치한다.

- Claude Code Panel 레이아웃: 타임라인 → **입력창** → 터미널 (위에서 아래로)
- 입력창은 타임라인 영역과 터미널 영역 사이에 고정 배치
- 리사이즈 핸들은 입력창 위(타임라인↔입력창)가 아닌, 기존 위치(입력창↔터미널) 유지
- 입력창 높이: 기본 1줄, 내용에 따라 자동 확장 (최대 높이 제한)

### REQ-2: 텍스트 입력 및 전송

입력 후 Enter를 누르면 하단 터미널에 텍스트를 전송한다.

- Enter 키: 입력 텍스트를 하단 터미널(tmux 세션)에 전송
- 전송 방식: tmux send-keys로 텍스트 + Enter 전달
- 전송 후 입력창 클리어
- 빈 입력 상태에서 Enter: 무시 (빈 문자열 전송 안 함)

### REQ-3: 여러 줄 입력

Shift+Enter로 줄바꿈하여 여러 줄을 입력할 수 있다.

- Shift+Enter: 줄바꿈 삽입
- 줄바꿈 시 입력창 높이 자동 확장
- 여러 줄 텍스트 전송 시: 줄바꿈을 포함하여 터미널에 전달
  - Claude Code CLI가 여러 줄 입력을 어떻게 처리하는지에 따라 전송 방식 결정
  - 줄 단위 send-keys 또는 전체 텍스트 붙여넣기 방식

### REQ-4: 입력창 포커스 단축키

단축키로 입력창에 빠르게 포커스를 이동할 수 있다.

- 단축키: 기존 cmux 호환 단축키 체계에 추가
- 입력창 포커스 시 커서가 입력 영역에 위치
- 터미널에서 입력창으로, 입력창에서 터미널로 포커스 이동 가능
- Escape: 입력창에서 포커스 해제 (터미널로 포커스 이동)

### REQ-5: 조건부 표시

Claude Code가 비활성일 때는 입력창을 숨긴다.

- Claude Code 활성 (panelType === 'claude-code' + 세션 active) → 입력창 표시
- Claude Code 비활성 (panelType === 'terminal' 또는 세션 none) → 입력창 숨김
- 세션 목록 뷰에서도 입력창 숨김 (세션 선택 전이므로)
- 표시/숨김 전환 시 부드러운 애니메이션

---

## 비기능 요구사항

### NFR-1: 입력 반응성

입력 텍스트가 터미널에 전달되기까지 체감 지연이 없어야 한다.

### NFR-2: 기존 기능 영향 없음

입력창 추가가 기존 타임라인, 터미널, 세션 탐색 기능에 영향을 주지 않아야 한다.

### NFR-3: 터미널 세션 무관

입력창은 텍스트를 tmux에 전달만 할 뿐, 터미널 세션 자체에 영향을 주지 않는다. 입력창 없이도 터미널에 직접 입력 가능 (기존 방식 유지).

## 범위 제외 (Phase 10에서 하지 않는 것)

| 항목 | 사유 |
|---|---|

| 자동 완성 / 제안 | 범위 과대 |

| 세션 메타 상단 바 | Phase 11 범위 |

## 기술 구성

### 입력 전송 흐름

```
사용자: 입력창에 텍스트 입력 → Enter
├── 클라이언트: WebSocket으로 입력 텍스트 전송
├── 서버: tmux send-keys -t {session} "{텍스트}" Enter
├── 터미널(tmux): Claude Code CLI에 텍스트 전달
└── Claude Code: 메시지 처리 → 타임라인에 반영
```

### 포커스 관리 흐름

```
타임라인/터미널 포커스 상태
├── 단축키 입력 → 입력창 포커스
├── Enter로 전송 → 입력창 포커스 유지 (연속 입력 가능)
├── Escape → 터미널로 포커스 이동
└── 터미널 클릭 → 터미널 포커스
```

## 검증 시나리오

1. **기본 입력**: 입력창에 텍스트 입력 → Enter → 터미널에 텍스트가 전달되고 Claude Code가 처리
2. **여러 줄 입력**: Shift+Enter로 줄바꿈 → 입력창 높이 확장 → Enter로 전체 전송
3. **빈 입력 무시**: 빈 상태에서 Enter → 아무 일도 안 일어남
4. **포커스 단축키**: 단축키 입력 → 입력창에 커서 이동 → 바로 타이핑 가능
5. **Escape**: 입력창에서 Escape → 터미널로 포커스 이동
6. **조건부 표시**: Claude Code 활성 시 입력창 표시, 비활성 시 숨김
7. **전송 후 클리어**: Enter로 전송 후 입력창이 비워짐
8. **터미널 직접 입력**: 입력창 무시하고 터미널에 직접 타이핑 → 정상 동작
9. **세션 목록 뷰**: 세션 목록 표시 중에는 입력창 숨김
10. **기존 기능 호환**: 입력창 추가 후에도 타임라인, 터미널, 세션 탐색 등 모두 정상
