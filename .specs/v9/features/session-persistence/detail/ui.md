# 화면 구성

## 개요

session-persistence는 서버 측 영속화 + 자동 resume 기능이므로 전용 UI는 없다. 서버 재시작 후 자동 복원 과정에서 기존 Claude Code Panel UI가 자연스럽게 활용된다.

## 자동 resume 진행 중 UI

서버 재시작 후 Claude Code Panel이 마운트되면, 자동 resume이 진행되는 동안 기존 UI 상태를 활용한다.

### 자동 resume 대기 중

```
┌─ Claude Code Panel ──────────────────────┐
│  ┌─ 타임라인 영역 ────────────────────┐  │
│  │                                    │  │
│  │      ⟳ 이전 세션을 복원하는 중...   │  │
│  │                                    │  │
│  └────────────────────────────────────┘  │
│  ━━━━━━━━━━━ 리사이즈 핸들 ━━━━━━━━━━━━  │
│  ┌─ 터미널 영역 ──────────────────────┐  │
│  │  $ claude --resume abc123...       │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

- 타임라인 영역: `Loader2` 스피너 + "이전 세션을 복원하는 중..." 메시지
- 스타일: 중앙 정렬, `text-sm text-muted-foreground`
- 하단 터미널: resume 명령어가 실행되는 모습이 보임

### 자동 resume 성공

- 스피너 제거 → 타임라인 렌더링 (Phase 8 동작)
- 별도 성공 알림 없음 (타임라인 표시 자체가 피드백)

### 자동 resume 실패

- 스피너 제거 → 세션 목록 뷰로 전환
- 터미널에 CLI 에러 메시지가 표시됨
- 별도 토스트 없음 (터미널 출력이 피드백)
