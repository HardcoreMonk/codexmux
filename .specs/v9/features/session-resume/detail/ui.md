# 화면 구성

## 개요

session-resume의 UI는 세션 목록 항목의 클릭 피드백과 결과 알림으로 구성된다. 주 화면은 session-list-ui에서 정의하고, 이 문서에서는 resume 과정의 인터랙션 피드백만 정의한다.

## resume 진행 중 피드백

### 클릭한 세션 항목

```
┌──────────────────────────────────────────┐
│    03/21 14:30              ⟳ 2시간 전   │  ← Loader2 animate-spin
│    "버그 수정해줘"                  12턴   │
└──────────────────────────────────────────┘
```

- 클릭한 항목: 좌측에 `Loader2` 스피너 (`animate-spin`, `size={14}`, `text-ui-purple`)
- 기존 `●` 마커 또는 빈 공간을 스피너로 대체
- 배경: `bg-ui-purple/5` (하이라이트)

### 비활성화된 다른 항목

- `opacity-50`
- `pointer-events-none`
- 중복 클릭 방지

## resume 결과 알림

### 성공

- 스피너 제거 → 타임라인 뷰로 전환
- 전환 시 별도 토스트 없음 (뷰 전환 자체가 피드백)

### 차단 (프로세스 실행 중)

- sonner 토스트: `toast.warning("터미널에서 다른 프로세스가 실행 중입니다")`
- 토스트 부가 설명: `"현재 실행 중인 프로세스: {processName}"`
- 세션 목록 상태 유지, 스피너 제거, 항목 재활성화

### 에러

- sonner 토스트: `toast.error("세션을 재개할 수 없습니다")`
- 세션 목록 상태 유지, 스피너 제거, 항목 재활성화
