# Phase 9 — Claude Code 세션 탐색 PRD

## 목표

과거 Claude Code 세션을 조회하고, 선택하여 `--resume`으로 이어서 작업할 수 있게 하는 것.

현재(Phase 8)는 터미널에서 `claude`를 실행해야 타임라인이 표시된다. Phase 9에서는 활성 세션이 없을 때도 과거 세션 목록을 탐색하고, 선택한 세션으로 바로 resume할 수 있다.

## 완료 조건

과거 Claude Code 세션을 탐색하고 선택하여 이어서 작업 가능.

---

## 현재 상태 (Phase 8 완료)

### 이미 구현된 것

- Claude Code Panel: 타임라인 뷰 + 터미널
- `claude` 명령어 실행 감지 → Panel 타입 자동 전환
- 세션 파일 파싱 → 타임라인 렌더링
- 세션 파일 실시간 감시 (fs.watch)
- Panel 타입 수동 전환 (Claude Code ↔ Terminal)
- 프로젝트 디렉토리 기반 세션 파일 매핑

### Phase 9에서 추가할 것

- 프로젝트별 과거 세션 목록 UI
- 세션 메타정보 표시 (시간, 요약 등)
- 세션 선택 → `--resume` 자동 연결
- 활성 세션이 없을 때 세션 목록 표시
- Surface별 Claude Code 세션 ID 영속화 및 서버 재시작 시 자동 resume

## 요구사항

### REQ-1: 세션 목록 조회

프로젝트별 Claude Code 과거 세션 목록을 조회한다.

- `~/.claude/projects/` 하위에서 현재 Workspace 프로젝트에 해당하는 세션 파일들을 탐색
- 세션 파일을 파싱하여 목록 데이터 구성
- 최신순(마지막 활동 시간 기준)으로 정렬
- 서버 API로 세션 목록 제공 (WebSocket 또는 REST)

### REQ-2: 세션 메타정보 표시

세션 목록에서 각 세션의 메타정보를 표시한다.

- 표시 항목:
  - 세션 시작 시간
  - 마지막 활동 시간
  - 첫 번째 사용자 메시지 (세션 요약 역할)
  - 대화 턴 수 (사용자 메시지 수)
- 세션 파일을 경량 파싱하여 메타정보 추출 (전체 타임라인을 로드하지 않음)

### REQ-3: 세션 목록 UI

Claude Code Panel에서 활성 세션이 없을 때 세션 목록을 표시한다.

- 타임라인 영역에 세션 목록을 렌더링
- 각 세션 항목: 시간, 요약, 대화 턴 수 등 메타정보 표시
- 세션 항목 클릭 → 해당 세션으로 resume (REQ-5)
- 스크롤 가능한 목록 (세션이 많을 경우)
- 활성 세션이 있을 때는 기존 타임라인 뷰 표시 (Phase 8과 동일)

### REQ-4: 세션 목록 ↔ 타임라인 전환

활성 세션 유무에 따라 세션 목록과 타임라인 뷰를 자동 전환한다.

- 활성 세션 없음 → 세션 목록 표시
- 세션 선택 또는 `claude` 실행 → 타임라인 뷰로 전환
- 활성 세션 종료 → 세션 목록으로 복귀
- 타임라인 뷰에서 세션 목록으로 돌아가는 UI 제공 (뒤로가기 등)

### REQ-5: `--resume` 자동 연결

세션 목록에서 세션 선택 시 해당 세션으로 `--resume` 연결한다.

- 세션 항목 클릭 → 하단 터미널에 `claude --resume {session_id}` 명령어 자동 입력 및 실행
- 터미널에 명령어를 전송하는 방식 (tmux send-keys 등)
- resume 후 타임라인 뷰로 자동 전환 (해당 세션의 타임라인 표시)
- resume 실패 시 (세션 파일 손상 등) 에러 표시

## 비기능 요구사항

### NFR-1: 세션 목록 로딩 성능

세션 목록 조회 시 체감 지연이 없어야 한다. 세션 파일이 많아도 목록 표시까지 빠르게 로딩되어야 한다.
- 메타정보만 경량 파싱 (전체 세션 파일을 읽지 않음)

### NFR-2: 기존 기능 호환

세션 탐색 기능 추가 후에도 기존 Claude Code Panel의 모든 기능이 정상 동작해야 한다.
- 자동 전환, 타임라인 렌더링, 실시간 감시, 수동 전환 등

### NFR-3: 영속성

세션 탐색 관련 상태는 별도 영속화가 필요하지 않다.
- 세션 목록은 `~/.claude/projects/` 파일 시스템에서 동적으로 조회
- 마지막 선택 세션 등은 저장하지 않음 (항상 최신 목록 표시)

## 범위 제외 (Phase 9에서 하지 않는 것)

| 항목 | 사유 |
|---|---|
| 세션 삭제 | Claude Code CLI가 관리하는 파일이므로 직접 조작하지 않음 |
| 세션 검색/필터 | 추후 필요 시 구현 |
| 세션 태그/북마크 | 추후 필요 시 구현 |
| 세션 간 비교 (diff) | 범위 과대 |
| 웹 입력창 (채팅 UI) | Phase 10 범위 |
| 새 세션 시작 버튼 | 터미널에서 `claude` 명령어로 시작 (기존 방식) |

## 기술 구성

### 세션 목록 조회 흐름

```
클라이언트: 세션 목록 요청
├── 서버: Workspace 프로젝트 디렉토리 확인
├── 서버: ~/.claude/projects/ 하위 세션 파일 탐색
├── 서버: 각 세션 파일 메타정보 경량 파싱
├── 서버: 최신순 정렬 → 클라이언트에 전송
└── 클라이언트: 세션 목록 UI 렌더링
```

### resume 연결 흐름

```
세션 항목 클릭
├── 클라이언트: resume 요청 (session_id)
├── 서버: tmux send-keys로 `claude --resume {session_id}` 전송
├── 서버: 세션 파일 fs.watch 시작
├── 클라이언트: 타임라인 뷰로 전환
└── 클라이언트: 타임라인 데이터 수신 → UI 업데이트
```

## 검증 시나리오

1. **세션 목록 표시**: Claude Code Panel에서 활성 세션 없음 → 과거 세션 목록이 최신순으로 표시
2. **메타정보 확인**: 세션 목록의 각 항목에 시간, 첫 메시지, 대화 턴 수가 표시
3. **세션 resume**: 세션 항목 클릭 → 터미널에 `claude --resume` 실행, 타임라인 뷰로 전환
4. **타임라인 전환**: resume 후 해당 세션의 타임라인이 정상 표시, 실시간 업데이트 동작
5. **세션 목록 복귀**: 타임라인 뷰에서 뒤로가기 → 세션 목록으로 복귀
6. **자동 전환**: 터미널에서 직접 `claude` 입력 → 세션 목록 대신 타임라인 뷰로 바로 전환
7. **세션 종료 후 복귀**: Claude Code 세션 종료 → 세션 목록으로 자동 복귀
8. **빈 세션 목록**: 프로젝트에 과거 세션이 없는 경우 → 빈 상태 메시지 표시
9. **다수 세션**: 세션이 많은 프로젝트 → 목록 스크롤 정상, 로딩 지연 없음
10. **기존 기능 호환**: 세션 탐색 추가 후에도 Phase 8의 자동 전환, 타임라인, 실시간 감시 등 모두 정상
