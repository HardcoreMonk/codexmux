# 화면 구성

## 개요

이 feature는 서버 내부 리팩토링이므로 UI 변경이 없다. API 응답 구조가 동일하게 유지되어 클라이언트 렌더링에 영향 없음.

## 변경 없는 화면

### 사이드바

- Workspace 목록, 활성 Workspace 표시, 접기/펼치기 — 모두 기존 동작 유지
- 로딩 스켈레톤 (3개 줄 pulse), 에러 상태 (AlertTriangle + 재시도), 빈 상태 — 기존 동작 유지

### 메인 영역

- Pane 레이아웃, 탭 바, 터미널 — 모두 기존 동작 유지
- Workspace 전환 시 fade out/in 애니메이션, 로딩 스피너 — 기존 동작 유지

## 체감 차이

| 항목 | Phase 5 | Phase 6 |
|---|---|---|
| Workspace 생성/삭제 후 서버 반영 | 300ms 디바운스 후 파일 저장 | 즉시 파일 저장 |
| 서버 비정상 종료 후 복원 | 디바운스 미반영분 유실 가능 | 완전 복원 |
| 체감 속도 | 동일 | 동일 (API 응답 시간 차이 무시 가능) |
