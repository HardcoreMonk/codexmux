# 화면 구성

## 개요

실시간 세션 감시는 서버 측 인프라이므로 직접적인 UI 요소가 없다. WebSocket 연결 상태에 따른 클라이언트 표시를 정의한다.

## WebSocket 연결 상태

### 연결 중 (connecting)

```
┌─ 타임라인 영역 ──────────────────────────┐
│                                          │
│  ┌─ 스켈레톤 ─────────────────────────┐  │
│  │ ░░░░░░░░░░░░░░░░  ← 메시지 블록    │  │
│  │ ░░░░░░  ← 도구 호출                │  │
│  │ ░░░░░░░░░░░░  ← 응답 블록          │  │
│  └────────────────────────────────────┘  │
│                                          │
└──────────────────────────────────────────┘
```

- 3개의 스켈레톤 블록으로 타임라인 형태 힌트
- `animate-pulse` 적용

### 연결됨 (connected)

- 정상 타임라인 표시
- 연결 상태 인디케이터 표시하지 않음 (타임라인이 업데이트되면 연결된 것)

### 재연결 중 (reconnecting)

```
┌─ 타임라인 영역 ──────────────────────────┐
│  (기존 타임라인 데이터 유지)               │
│  ...                                     │
│  ├─ 14:32 "수정 완료했습니다"              │
│                                          │
│  ┌─ 재연결 배너 ──────────────────────┐  │
│  │ ○ 연결이 끊어졌습니다. 재연결 중...  │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

- 기존 타임라인 데이터는 유지 (제거하지 않음)
- 하단에 인라인 배너 표시: `text-muted-foreground`, `bg-muted` 배경
- 재연결 성공 → 배너 제거 + 누락된 데이터 보충

### 연결 실패 (disconnected)

```
┌─ 타임라인 영역 ──────────────────────────┐
│  (기존 타임라인 데이터 유지)               │
│  ...                                     │
│                                          │
│  ┌─ 에러 배너 ────────────────────────┐  │
│  │ ✕ 연결 실패       [다시 시도]       │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

- "다시 시도" 버튼 제공
- 기존 데이터는 계속 표시

## 타임라인 업데이트 시각 효과

### 새 엔트리 추가

- 새 항목이 하단에 추가될 때 `fadeIn` 애니메이션 (150ms ease-out)
- 자동 스크롤 활성 시 부드러운 스크롤로 하단 이동

### 세션 전환

- 기존 타임라인 `fadeOut` (100ms)
- 스켈레톤 로딩 표시 (짧은 순간)
- 새 타임라인 `fadeIn` (150ms)
