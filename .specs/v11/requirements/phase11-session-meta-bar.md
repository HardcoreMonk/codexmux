# Phase 11 — 세션 메타 상단 바 PRD

## 목표

Claude Code Panel 상단에 현재 세션의 메타데이터를 한눈에 표시하는 것.

현재(Phase 10까지)는 세션의 제목, 시간, 토큰 사용량 등 메타정보를 확인하려면 세션 파일을 직접 열어봐야 한다. Phase 11에서는 타임라인 상단에 컴팩트한 메타 바를 배치하여, 세션 진행 중 핵심 정보를 실시간으로 확인할 수 있다.

## 완료 조건

Claude Code 세션 진행 중 상단 바에서 제목, 시간, 메시지 수, 브랜치, 토큰 사용량을 실시간으로 확인 가능.

---

## 현재 상태 (Phase 10 완료)

### 이미 구현된 것

- Claude Code Panel: 타임라인 뷰 + 입력창 + 축소 터미널
- 세션 파일 파싱 (JSONL → 타임라인 엔트리)
- 세션 파일 실시간 감시 (fs.watch + WebSocket)
- CLI 상태 감지 (idle/busy/inactive)
- 세션 목록 탐색 + resume 연결

### Phase 11에서 추가할 것

- 타임라인 상단에 세션 메타데이터 바 배치
- 세션 파일에서 메타데이터 파싱 (제목, 시간, 메시지 수, 토큰)
- git 브랜치 정보 표시
- 실시간 갱신
- 컴팩트 레이아웃 (한 줄에 핵심, 호버/클릭 시 상세)

---

## 요구사항

### REQ-1: 메타 바 배치

Claude Code Panel 타임라인 영역 상단에 메타 바를 배치한다.

- 위치: 타임라인 영역 최상단 (세션 목록 네비게이션 아래, 타임라인 엔트리 위)
- 고정 높이, 스크롤 영역 밖 (타임라인 스크롤 시 메타 바는 고정)
- 컴팩트 레이아웃: 한 줄에 핵심 정보 나열

### REQ-2: 표시 항목

메타 바에 다음 정보를 표시한다.

| 항목 | 데이터 소스 | 표시 형태 |
|---|---|---|
| 세션 제목 | 첫 사용자 메시지 또는 세션 파일 내 summary 필드 | 텍스트 (truncate) |
| 생성 시간 | 세션 파일 첫 엔트리 타임스탬프 | 상대 시간 ("2시간 전") |
| 최종 수정 시간 | 세션 파일 마지막 엔트리 타임스탬프 | 상대 시간 |
| 총 메시지 수 | user-message + assistant-message 엔트리 카운트 | "사용자 12 / 어시스턴트 15" |
| 현재 git 브랜치 | 프로젝트 디렉토리의 git HEAD | 브랜치명 (아이콘 + 텍스트) |
| 토큰 사용량 | 각 메시지의 usage 필드 집계 | "입력 50K / 출력 30K / 총 80K" |

### REQ-3: 세션 파일 메타데이터 파싱

세션 파일(JSONL)에서 메타데이터를 추출한다.

- **제목**: 세션 파일 내 summary 필드가 있으면 사용, 없으면 첫 사용자 메시지를 제목으로 사용
- **토큰 사용량**: 각 메시지의 usage 필드(`input_tokens`, `output_tokens`)를 집계
  - 기존 세션 파서를 확장하여 usage 정보 추출
- **메시지 수**: user-message, assistant-message 엔트리 각각 카운트
- **시간**: 첫/마지막 엔트리의 타임스탬프

### REQ-4: git 브랜치 정보

현재 프로젝트의 git 브랜치를 표시한다.

- Panel의 tmux 세션 cwd 기준으로 git 브랜치 조회
- `git rev-parse --abbrev-ref HEAD` 또는 `.git/HEAD` 파일 읽기
- 브랜치 변경 시 실시간 갱신 (주기적 폴링 또는 파일 감시)

### REQ-5: 실시간 갱신

세션 파일 변경 시 메타 바 정보를 실시간으로 업데이트한다.

- 기존 fs.watch 인프라(timeline:append)를 활용
- 새 엔트리 추가 시 메시지 수, 토큰 사용량, 최종 수정 시간 자동 갱신
- git 브랜치는 별도 주기 갱신 (세션 파일과 무관)

### REQ-6: 컴팩트/상세 레이아웃

메타 바는 기본적으로 컴팩트하게 표시하고, 필요 시 상세 정보를 확인할 수 있다.

- **컴팩트** (기본): 한 줄에 제목 + 시간 + 메시지 수 + 토큰 요약
- **상세** (호버 또는 클릭): 확장하여 전체 정보 표시 (생성/수정 시간, 입력/출력 토큰 분리, git 브랜치 등)

---

## 비기능 요구사항

### NFR-1: 실시간성

새 엔트리 추가 시 메타 바 정보 업데이트까지 체감 지연이 없어야 한다.

### NFR-2: 기존 기능 영향 없음

메타 바 추가가 타임라인, 입력창, 터미널, 세션 탐색 기능에 영향을 주지 않아야 한다.

### NFR-3: 성능

토큰 집계는 타임라인 엔트리가 추가될 때마다 증분 계산한다 (전체 재계산 아님).

---

## 범위 제외 (Phase 11에서 하지 않는 것)

| 항목 | 사유 |
|---|---|
| 토큰 비용 계산 (달러 환산) | 모델 가격이 변동되므로 범위 제외 |
| 세션 간 토큰 비교 | 범위 과대 |
| 메타 바 커스터마이징 (표시 항목 선택) | 추후 필요 시 구현 |
| 세션 제목 편집 | Claude Code CLI 세션 파일은 읽기 전용 |

## 기술 구성

### 메타데이터 수집 흐름

```
세션 파일 변경 (fs.watch)
├── timeline:append로 새 엔트리 수신
├── 클라이언트: 증분 업데이트
│   ├── 메시지 수 += 1 (타입에 따라)
│   ├── 토큰 사용량 += usage (있으면)
│   └── 최종 수정 시간 = 새 엔트리 타임스탬프
└── 메타 바 UI 갱신
```

### git 브랜치 조회 흐름

```
Panel 마운트 시
├── tmux 세션 cwd 조회
├── cwd에서 git branch 조회
├── 메타 바에 표시
└── 주기적 갱신 (30초 간격)
```

## 검증 시나리오

1. **메타 바 표시**: Claude Code 세션 활성 시 타임라인 상단에 메타 바가 표시됨
2. **제목 표시**: 첫 사용자 메시지가 세션 제목으로 표시됨
3. **실시간 갱신**: Claude가 응답할 때마다 메시지 수, 토큰 사용량이 즉시 업데이트
4. **git 브랜치**: 현재 프로젝트의 git 브랜치가 정확히 표시됨
5. **컴팩트/상세 전환**: 메타 바 클릭 시 상세 정보가 확장되어 표시됨
6. **토큰 집계**: 입력/출력 토큰이 정확히 집계되어 표시됨
7. **세션 전환**: 다른 세션으로 resume 시 메타 바가 새 세션 정보로 갱신됨
8. **세션 없음**: 세션이 없을 때(세션 목록 뷰) 메타 바 숨김
9. **대용량 세션**: 긴 대화 세션에서도 메타 바 렌더링 지연 없음
10. **기존 기능 호환**: 메타 바 추가 후에도 타임라인, 입력창, 터미널, 세션 탐색 모두 정상
