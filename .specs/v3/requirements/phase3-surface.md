# Phase 3 — Surface (탭) PRD

## 목표

하나의 화면에서 여러 터미널 탭을 관리하는 것.

Phase 2에서 구현한 tmux 기반 단일 터미널을 확장하여, 탭 바 UI를 통해 여러 독립 터미널 세션을 생성/전환/삭제할 수 있게 한다. 각 탭은 독립된 tmux 세션과 1:1로 매핑된다.

## 완료 조건

탭으로 여러 터미널을 관리할 수 있다. 탭 생성, 전환, 삭제가 가능하며 각 탭은 독립된 터미널 세션이다.

---

## 현재 상태 (Phase 2 완료)

```
┌─────────────────────────────────────────┐
│                                         │
│           단일 터미널 (전체 화면)          │
│           tmux 세션 1개                  │
│                                         │
└─────────────────────────────────────────┘
```

- 하나의 tmux 세션 (`pt-{wId}-{pId}-{sId}`)
- 세션 매칭: 첫 번째 `pt-*` 세션에 attach
- 전체 화면 터미널 (탭 없음)

## 목표 상태 (Phase 3)

```
┌─ Tab 1 ──┬─ Tab 2 ──┬─ Tab 3 ──┬─ + ─┐
│                                        │
│          활성 탭의 터미널                 │
│          (각 탭 = 독립 tmux 세션)        │
│                                        │
└────────────────────────────────────────┘
```

- 여러 tmux 세션을 탭으로 관리
- 탭 바 UI로 생성/전환/삭제
- 활성 탭의 터미널만 화면에 표시

---

## 요구사항

### REQ-1: 탭 바 UI

터미널 영역 상단에 탭 바를 표시한다.

- 화면 상단에 탭 바, 나머지 영역에 활성 탭의 터미널을 렌더링한다
- 각 탭에 탭 이름(기본: "Terminal 1", "Terminal 2", ...)과 닫기 버튼을 표시한다
- 탭 바 우측에 새 탭 추가 버튼(+)을 배치한다
- 활성 탭은 시각적으로 구분한다 (배경색, 하단 보더 등)
- 탭이 1개일 때도 탭 바를 표시한다 (일관된 UI)
- 탭 개수가 많아지면 탭 바가 가로 스크롤되거나 축소된다

### REQ-2: 탭 생성

새 탭을 생성하면 독립된 tmux 세션이 시작된다.

- 탭 추가 버튼(+) 클릭 또는 단축키로 새 탭을 생성한다
- 새 탭 생성 시 새 tmux 세션(`pt-{wId}-{pId}-{sId}`, nanoid)을 생성한다
- 새 탭은 탭 바 맨 우측에 추가되고 자동으로 활성화된다
- 새 탭의 터미널은 사용자의 기본 쉘로 시작한다

### REQ-3: 탭 전환

탭 클릭으로 활성 터미널을 전환한다.

- 탭 클릭 시 해당 탭의 터미널로 전환한다
- 전환 시 현재 WebSocket을 닫고(detach), 새 탭의 세션 ID로 새 WebSocket을 연결한다 (`/api/terminal?session={id}`)
- 단일 xterm.js 인스턴스를 재활용한다 — 전환 시 `xterm.reset()` 또는 `xterm.clear()` 후 새 세션 데이터 수신
- tmux attach 시 자동 redraw로 해당 세션의 화면이 복원된다
- 비활성 탭의 tmux 세션은 백그라운드에서 계속 실행된다 (프로세스 유지)

### REQ-4: 탭 삭제

탭을 닫으면 해당 tmux 세션이 종료된다.

- 탭의 닫기 버튼(×) 클릭으로 탭을 삭제한다
- 삭제 시 해당 tmux 세션을 `kill-session`으로 종료한다
- 활성 탭을 닫으면 인접한 탭으로 자동 전환한다 (우측 우선, 없으면 좌측)
- 마지막 탭을 닫으면 새 탭을 자동 생성한다 (빈 상태 방지)
- 탭 내에서 `exit` 실행 시에도 해당 탭이 닫힌다 (Phase 2 세션 종료 정책 확장)

### REQ-5: 탭-세션 매핑

클라이언트가 어떤 세션에 연결할지를 관리한다.

- Phase 2의 "첫 번째 pt-* 세션에 attach" 방식에서, 클라이언트가 세션 ID를 URL 쿼리 파라미터로 지정하는 방식으로 전환한다
- WebSocket 연결: `/api/terminal?session=pt-a1b2c3-d4e5f6-g7h8i9` (기존 세션) 또는 `/api/terminal` (새 세션 생성)
- 기존 바이너리 메시지 프로토콜(0x00~0x03)은 변경 없음
- `server.ts`의 upgrade 핸들러에서 `request.url`의 `session` 파라미터를 파싱하여 세션 매칭
- Phase 2 하위 호환: `session` 파라미터 없으면 기존 동작 (첫 번째 pt-* 세션 또는 새 세션)

### REQ-6: 탭 영속성

서버 재시작/새로고침 후에도 탭 목록이 복원된다.

- 탭 목록(이름, 순서, 활성 탭, 세션 ID)을 `~/.purple-terminal/tabs.json`에 저장한다
- 탭 생성/삭제/이름 변경/순서 변경 시 자동 저장 (디바운스 적용)
- 서버 시작 시 `tabs.json` + tmux 세션 목록을 크로스 체크하여 복원한다
  - tabs.json에 있지만 tmux 세션이 없는 탭 → 제거
  - tmux 세션이 있지만 tabs.json에 없는 세션 → 탭으로 추가 (orphan 복구)
- 새로고침 시 서버에서 탭 목록을 조회하여 복원한다
- 탭이 복원되면 마지막 활성 탭이 자동으로 선택된다

### REQ-7: 탭 순서 변경

탭 드래그 앤 드롭으로 순서를 변경할 수 있다.

- 탭을 드래그하여 원하는 위치에 놓으면 순서가 변경된다
- 변경된 순서는 `tabs.json`에 즉시 반영된다

### REQ-8: 탭 이름 변경

탭 이름을 사용자가 변경할 수 있다.

- 탭 더블클릭으로 인라인 편집 모드 진입
- 이름 확정(Enter/blur) 시 `tabs.json`에 저장
- 기본 이름: "Terminal 1", "Terminal 2", ... (생성 순서 기반)

---

## 비기능 요구사항

### NFR-1: 탭 전환 속도

탭 전환이 즉각적으로 느껴져야 한다. tmux detach → attach 과정이 사용자에게 지연으로 체감되지 않아야 한다.

### NFR-2: 백그라운드 세션 유지

비활성 탭의 tmux 세션에서 실행 중인 프로세스(빌드, 테스트 등)가 중단되지 않아야 한다.

### NFR-3: 탭 바 최소 침범

탭 바가 터미널 영역을 최소한으로 줄여야 한다. 높이를 낮게 유지하고, Muted 팔레트에 맞는 디자인을 적용한다.

### NFR-4: Phase 2 호환

Phase 2의 세션 영속성(서버 재시작 복원, detaching 플래그, close code 정책)이 탭 환경에서도 동일하게 동작해야 한다.


### Phase 2 대비 주요 변경

| 항목 | Phase 2 | Phase 3 |
|---|---|---|
| tmux 세션 수 | 1개 | 탭 수만큼 N개 |
| 세션 매칭 | 첫 번째 `pt-*` 세션 | 클라이언트가 세션 ID 지정 |
| WebSocket | 단일 연결 | 단일 연결 유지, 탭 전환 시 끊고 재연결 |
| UI | 전체 화면 터미널 | 탭 바 + 터미널 영역 |
| 레이아웃 | 100vh | 탭 바 높이 제외한 나머지 |

## 검증 시나리오

1. **탭 생성**: + 버튼 클릭 시 새 탭이 생성되고 새 터미널이 시작된다
2. **탭 전환**: 탭 클릭 시 해당 터미널로 즉시 전환된다. 이전 탭의 프로세스는 계속 실행된다
3. **탭 삭제**: × 버튼 클릭 시 탭이 닫히고 tmux 세션이 종료된다
4. **마지막 탭 삭제**: 마지막 탭을 닫으면 새 탭이 자동 생성된다
5. **탭 내 exit**: 터미널에서 exit 실행 시 해당 탭이 닫힌다
6. **서버 재시작 복원**: 서버 재시작 후 모든 탭과 터미널 상태가 복원된다
7. **새로고침 복원**: 브라우저 새로고침 후 탭 목록과 활성 탭이 복원된다
8. **백그라운드 실행**: Tab 1에서 빌드 실행 중 Tab 2로 전환 → Tab 1로 돌아오면 빌드가 계속 진행 중이다
9. **다중 탭 동시 프로세스**: 여러 탭에서 각각 다른 작업 실행 중 서버를 재시작해도 모두 복원된다
10. **탭 바 UI**: 탭이 많아져도 UI가 깨지지 않는다 (스크롤 또는 축소)
11. **탭 순서 변경**: 탭 드래그로 순서를 변경하면 재시작 후에도 순서가 유지된다
12. **탭 이름 변경**: 탭 더블클릭으로 이름을 변경하면 재시작 후에도 이름이 유지된다

## 확정된 결정사항

| 항목 | 결정 | 근거 |
|---|---|---|
| WebSocket 연결 전략 | 단일 연결, 탭 전환 시 끊고 재연결 | 리소스 효율적, 서버 부하 최소화 |
| 탭 상태 저장 위치 | 서버 `~/.purple-terminal/tabs.json` | 다른 브라우저에서도 복원 가능 |
| xterm.js 인스턴스 관리 | 단일 인스턴스 재활용 | 메모리 절약, tmux redraw로 화면 복원 |
| 탭 순서/이름 변경 | Phase 3 범위에 포함 | 드래그 재정렬 + 더블클릭 이름 편집 |
| 세션 ID 전달 방식 | URL 쿼리 파라미터 (`?session={id}`) | 프로토콜 변경 없음, Phase 2 하위 호환, 디버깅 용이 |
