# 탭 UI

## 개요

Phase 2의 전체 화면 단일 터미널을 탭 기반 다중 터미널 UI로 확장한다. 화면 상단에 탭 바를 배치하고, 나머지 영역에 활성 탭의 터미널을 렌더링한다. 탭 생성/전환/삭제/순서 변경/이름 변경을 지원하며, 단일 xterm.js 인스턴스를 재활용하여 메모리를 절약한다.

## 주요 기능

### 탭 바

- 화면 최상단에 탭 바 배치, 나머지 영역에 터미널
- Muted 팔레트 다크 테마 적용 — 터미널 배경과 자연스럽게 이어지는 톤
- 탭 바 높이 최소화 (28~32px) — 터미널 영역 침범 최소
- 활성 탭: 밝은 배경 또는 하단 보더로 시각적 구분
- 비활성 탭: 터미널 배경보다 약간 밝은 톤, hover 시 미세한 밝기 변화
- 탭이 1개일 때도 탭 바 표시 (일관된 UI, 추가 버튼 접근성)
- 탭 바 우측에 + 버튼 (새 탭 추가)

### 탭 오버플로우

- 탭 개수가 탭 바 너비를 초과하면 가로 스크롤 활성화
- 스크롤 시 좌우 화살표 버튼 또는 마우스 휠 스크롤
- 활성 탭이 뷰포트 밖에 있으면 자동 스크롤하여 노출

### 탭 생성

- + 버튼 클릭으로 새 탭 생성
- 서버에 탭 생성 요청 (POST `/api/tabs`) → 새 tmux 세션 생성 → 세션 ID 반환
- 반환된 세션 ID로 WebSocket 연결 (`/api/terminal?session={id}`)
- 새 탭은 맨 우측에 추가되고 자동으로 활성화
- 기본 이름: "Terminal {N}" (현재 탭 중 최대 번호 + 1)
- 생성 중 + 버튼에 미세한 로딩 피드백 (disabled 상태, 스피너 등)

### 탭 전환

- 탭 클릭 시 해당 탭으로 전환
- 전환 흐름:
  1. 클릭한 탭을 활성 상태로 즉시 UI 반영 (optimistic)
  2. 현재 WebSocket 닫기 (detach)
  3. xterm.js 화면 초기화 (`reset()`)
  4. 새 세션 ID로 WebSocket 연결 (`/api/terminal?session={id}`)
  5. tmux attach → 자동 redraw → 이전 화면 복원
- 전환 중 터미널 영역에 미세한 로딩 인디케이터 (connecting 상태 재활용)
- 비활성 탭의 tmux 세션은 백그라운드 유지 (프로세스 중단 없음)

### 탭 삭제

- 각 탭의 닫기 버튼(×) 클릭으로 삭제
- × 버튼은 hover 시에만 표시 (활성 탭은 항상 표시)
- 활성 탭 삭제 시 인접 탭으로 자동 전환 (우측 우선, 없으면 좌측)
- 마지막 탭 삭제 시 새 탭 자동 생성 (빈 상태 방지)
- 삭제 시 서버에 요청 (DELETE `/api/tabs/{id}`) → tmux 세션 kill
- 탭 내 `exit` 실행 시에도 해당 탭 자동 삭제 (close code 1000 수신 → 탭 제거 → 인접 탭 전환)

### 탭 순서 변경 (드래그 앤 드롭)

- 탭을 드래그하여 원하는 위치에 드롭
- 드래그 중 드롭 위치 시각적 인디케이터 (삽입 위치 표시)
- 드롭 시 서버에 순서 업데이트 요청 (PATCH `/api/tabs/order`)
- + 버튼은 드래그 대상에서 제외 (항상 맨 우측 고정)

### 탭 이름 변경

- 탭 더블클릭으로 인라인 편집 모드 진입
- 인라인 input이 탭 이름 위치에 표시 (기존 이름이 선택된 상태)
- Enter 또는 blur 시 이름 확정 → 서버에 업데이트 (PATCH `/api/tabs/{id}`)
- Escape 시 편집 취소 (이전 이름 복원)
- 빈 이름 입력 시 기본 이름으로 복원

### 단일 xterm.js 인스턴스 재활용

- 탭마다 별도 xterm.js를 생성하지 않고, 하나의 인스턴스를 재활용
- 탭 전환 시: `xterm.reset()` → 새 WebSocket 연결 → tmux redraw로 화면 복원
- 메모리 효율적이지만, 전환 시 잠깐 빈 화면이 보일 수 있음 (tmux attach 속도에 의존)
- Phase 2에서 구현한 ResizeObserver, addon-fit, addon-webgl 등은 그대로 유지

### 페이지 로드 시 탭 복원

- 페이지 로드 시 `/api/tabs`로 탭 목록 조회
- 탭 목록이 비어있으면 새 탭 자동 생성
- 마지막 활성 탭의 세션에 WebSocket 연결
- 새로고침 후에도 탭 목록 + 활성 탭이 복원
