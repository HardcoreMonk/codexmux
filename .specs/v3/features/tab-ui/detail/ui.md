# 화면 구성

## 전체 레이아웃

```
┌─ Tab 1 ──┬─ Tab 2 ──┬─ Tab 3* ─┬──────────────┬─ + ─┐  ← 탭 바 (28~32px)
├────────────────────────────────────────────────────────┤
│                                                        │
│                                                        │
│                 xterm.js 터미널                          │  ← 나머지 영역 전체
│                 (활성 탭의 세션)                          │
│                                                        │
│                                                        │
│                                                [상태]  │  ← 연결 상태 오버레이 (우하단)
└────────────────────────────────────────────────────────┘
```

- 탭 바: 상단 고정, 28~32px 높이
- 터미널 영역: `calc(100vh - 탭 바 높이)`, 풀 너비
- 연결 상태 오버레이: Phase 2와 동일 (터미널 영역 내 absolute)

## 컴포넌트 구조

```
pages/index.tsx
└── <TerminalPage />
    ├── <TabBar />                        ← 신규
    │   ├── <Tab />                       ← 탭 항목 (반복)
    │   │   ├── 탭 이름 (span 또는 inline input)
    │   │   └── 닫기 버튼 (×)
    │   └── <AddTabButton />              ← + 버튼
    ├── <TerminalContainer />             ← Phase 2 재활용
    │   └── xterm.js Terminal 인스턴스
    ├── <ConnectionStatus />              ← Phase 2 재활용
    └── <SessionEndedOverlay />           ← Phase 2 재활용 (탭 삭제로 확장)
```

## 탭 바 (`TabBar`)

| 속성 | 값 | 설명 |
|---|---|---|
| 높이 | 28~32px | 터미널 영역 침범 최소화 |
| 배경 | Zinc 다크 계열 (`oklch(0.18 0.006 286)` 수준) | 터미널보다 약간 어두운 톤 |
| 보더 | 하단 1px, Zinc 600 수준 | 탭 바와 터미널 경계 |
| overflow-x | `auto` (스크롤) | 탭 오버플로우 시 가로 스크롤 |
| 스크롤바 | 숨김 (`scrollbar-width: none`) | 시각적 노이즈 제거 |

### 탭 오버플로우 처리

```
┌─◀─┬─ Tab 1 ─┬─ Tab 2 ─┬─ Tab 3 ─┬─ ... ─┬─▶─┬─ + ─┐
```

- 탭이 너비를 초과하면 좌우 화살표 버튼 표시
- 마우스 휠로 가로 스크롤 가능
- 활성 탭이 뷰포트 밖이면 `scrollIntoView({ behavior: 'smooth', inline: 'nearest' })`

## 탭 항목 (`Tab`)

### 상태별 스타일

| 상태 | 배경 | 텍스트 | 보더 | 닫기 버튼 |
|---|---|---|---|---|
| 활성 | 터미널 배경과 동일 (`oklch(0.21 0.006 286)`) | Zinc 밝은 톤 | 하단 2px `ui-blue` | 항상 표시 |
| 비활성 | 탭 바 배경과 동일 | Zinc 400 수준 | 없음 | hover 시 표시 |
| hover | 약간 밝은 톤 (`oklch(0.24 0.006 286)`) | Zinc 밝은 톤 | 없음 | 표시 |
| 드래그 중 | 반투명 (`opacity: 0.7`) | — | — | — |

### 탭 크기

| 속성 | 값 |
|---|---|
| 최소 너비 | 80px |
| 최대 너비 | 180px |
| 패딩 | `0 12px` |
| 이름 | `text-ellipsis overflow-hidden whitespace-nowrap` |
| 폰트 | `text-xs` (12px) |

### 닫기 버튼 (×)

| 속성 | 값 |
|---|---|
| 크기 | 16×16px |
| 아이콘 | lucide `X` (12px) |
| hover | 배경 Zinc 600, 원형 |
| 위치 | 탭 우측, 이름과 4px 간격 |

## 새 탭 추가 버튼 (`AddTabButton`)

| 속성 | 값 |
|---|---|
| 크기 | 28×28px |
| 아이콘 | lucide `Plus` (14px) |
| 위치 | 탭 바 맨 우측, 고정 (스크롤 영향 안 받음) |
| hover | 배경 Zinc 700, 원형 |
| disabled (생성 중) | `opacity: 0.5`, 클릭 불가 |

## 탭 이름 인라인 편집

```
더블클릭 전: ┌─ Terminal 1  × ─┐
더블클릭 후: ┌─ [Terminal 1   ] ─┐  ← input, 기존 이름 선택 상태
```

| 속성 | 값 |
|---|---|
| input 배경 | 투명 |
| input 보더 | 하단 1px `ui-blue` |
| 폰트 | 탭 이름과 동일 (`text-xs`) |
| 동작 | Enter/blur → 확정, Escape → 취소 |

## 드래그 앤 드롭

- 드래그 시작: 탭에 `draggable` 속성, 또는 포인터 이벤트 기반 커스텀 드래그
- 드래그 중: 드래그 대상 반투명, 드롭 위치에 2px 파란색 세로 인디케이터
- 드롭: 순서 변경 → 서버 업데이트 → UI 반영
- + 버튼 위에 드롭 불가 (드롭 영역에서 제외)

## 상태별 UI

### 로딩 (탭 목록 조회 중)

```
┌──────────────────────────────────────────────────────┐
│  ▓▓▓▓▓▓  ▓▓▓▓▓▓  ▓▓▓▓▓▓  +                          │  ← 탭 스켈레톤
├──────────────────────────────────────────────────────┤
│                                                      │
│                    연결 중...                          │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- 페이지 로드 직후 탭 바에 스켈레톤 (3개 정도)
- 탭 목록 조회 완료 후 실제 탭으로 교체

### 빈 상태 (탭 없음)

- 발생하지 않음 — 탭이 0개면 자동으로 첫 탭 생성
- API 조회 시 빈 배열 → 즉시 POST `/api/tabs` → 첫 탭 생성

### 에러 (탭 목록 조회 실패)

```
┌──────────────────────────────────────────────────────┐
│  ⚠ 탭 목록을 불러올 수 없습니다.  [재시도]             │
├──────────────────────────────────────────────────────┤
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 탭 전환 중

- 탭 바: 클릭한 탭 즉시 활성 (optimistic)
- 터미널: 잠깐 빈 화면 → connecting 인디케이터 → tmux redraw → 화면 복원
- 전체 과정이 200ms 이내로 느껴져야 함

## 접근성

- 탭 바: `role="tablist"`, 각 탭: `role="tab"`, `aria-selected`
- 터미널 영역: `role="tabpanel"`
- 키보드: Tab/Shift+Tab으로 탭 간 포커스 이동 (추후 단축키 Phase 7에서 확장)
- 닫기 버튼: `aria-label="탭 닫기"`
- 추가 버튼: `aria-label="새 탭"`
