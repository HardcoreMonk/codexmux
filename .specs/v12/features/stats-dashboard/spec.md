# 사용량 통계 대시보드

## 개요

Claude Code 사용량을 시각화하는 전용 페이지. `/stats` 라우트로 접근하며, 기존 터미널 레이아웃과 완전히 분리된다. 개요·토큰·활동 패턴·프로젝트별·세션 분석 5개 섹션과 기간 필터를 제공한다. 각 섹션은 Error Boundary로 감싸 파싱 실패 시 해당 섹션만 에러를 표시한다.

## 주요 기능

### 라우팅 및 네비게이션

- Next.js Pages Router: `pages/stats.tsx` — 별도 라우트
- 사이드바 하단에 통계 진입 메뉴 (lucide-react `BarChart3` 아이콘)
- 통계 페이지 진입 시 기존 터미널 WebSocket 연결은 끊김 (페이지 전환)
- 사이드바 Workspace 클릭 → `/` 라우트로 복귀, 터미널 재연결
- 사이드바는 통계 페이지에서도 표시 (Workspace 목록 + 통계 메뉴)

### 기간 필터

- 페이지 상단 고정: `오늘 | 7일 | 30일 | 전체` 탭 버튼 그룹
- 기본값: 30일
- 필터 변경 시 모든 섹션 데이터 갱신
- 1단계 데이터(stats-cache): 클라이언트 측 필터링 (즉시)
- 2~3단계 데이터: 서버 API에 기간 파라미터 전달하여 재조회

### 개요 섹션

- **숫자 카드 4개**: 총 세션 수, 총 메시지 수, 오늘 사용량, 이번 달 사용량
  - 각 카드: 지표명 (`text-xs text-muted-foreground`) + 숫자 (`text-2xl font-semibold`)
  - 전 기간 대비 변화율 표시 (예: "+12% vs 지난 달")
- **누적 추이 차트**: 에어리어 차트 (`--chart-1`)
  - X축: 날짜, Y축: 일별 세션 또는 메시지 수
  - 호버 시 Tooltip: 날짜 + 수치

### 토큰 사용량 섹션

- **모델별 토큰 바 차트**: 모델(Opus/Sonnet/Haiku)별 입력/출력/캐시 토큰
  - Opus → `var(--chart-5)`, Sonnet → `var(--chart-1)`, Haiku → `var(--chart-2)`
- **일별 토큰 추이**: 스택 에어리어 차트
  - 입력 → `var(--chart-1)`, 출력 → `var(--chart-2)`
- **모델별 비율**: 도넛 차트
  - 중앙에 총 토큰 수치 표시

### 활동 패턴 섹션

- **시간대별 히트맵**: 가로 52주 × 세로 7일 그리드 (GitHub 잔디 스타일)
  - 색상: `var(--chart-2)` (ui-teal) 계열, 0~4단계 강도
  - 호버: 날짜 + 사용량 Tooltip
- **일별 활동 차트**: 멀티 시리즈 바 차트 (메시지/세션/도구 호출)
- **요일별 패턴**: 요일별 평균 사용량 레이더 또는 바 차트

### 프로젝트별 분석 섹션

- **프로젝트별 수평 바 차트**: 토큰 소비 기준 내림차순
  - 프로젝트명: 디렉토리 경로 마지막 세그먼트
  - 색상: 범주형이므로 `--chart-5` (ui-purple), `--chart-3` (ui-coral), `--chart-6` (ui-pink) 등 중립 색상 사용
- **프로젝트별 세션/메시지 수**: 보조 지표

### 세션 분석 섹션

- **숫자 카드**: 평균 세션 길이, 최장 세션, 총 도구 호출 수
- **카테고리 분포**: facets 데이터 → 도넛 차트 (코드 작성, 리뷰, 문서 등)
- **목표 달성도**: facets 데이터 → 가로 바 차트 (달성/미달성)
- **history.jsonl 기반 지표**: 자주 사용하는 명령어 TOP 10, 입력 길이 분포 히스토그램

### 2단계 점진적 로딩

| 단계 | 시점 | 데이터 소스 | 렌더링 대상 |
|---|---|---|---|
| 1단계 | 페이지 진입 즉시 | stats-cache.json | 개요 카드, 토큰 추이, 히트맵 |
| 2단계 | 백그라운드 | 세션 JSONL | 프로젝트별 분석, 세션 상세 |
| 3단계 | 백그라운드 | facets + history | 카테고리 분포, 달성도, 명령어 순위 |

- 1단계: skeleton 없이 즉시 렌더링
- 2~3단계: 해당 섹션 skeleton UI → 데이터 도착 시 교체
- 파싱 진행률: 2단계 JSONL 파싱 시 진행률 바 표시 (대용량일 때)

### Error Boundary

- 각 섹션(개요/토큰/활동/프로젝트/세션)을 개별 Error Boundary로 래핑
- 파싱 실패 또는 데이터 구조 변경 시: 해당 섹션만 에러 UI 표시
- 에러 UI: "데이터를 불러올 수 없습니다" + 재시도 버튼
- 나머지 섹션은 정상 동작 유지

### 다크 모드

- 페이지 배경: `bg-background`
- 카드: `bg-card`, `border` (0.5px)
- 차트: recharts의 다크 모드 대응 (축/그리드 색상 `text-muted-foreground`)
- 히트맵: 다크 모드에서 강도 색상 조정
