# Phase 12 — 사용량 통계 페이지 PRD

## 목표

Claude Code 사용량을 시각적으로 분석하는 전용 통계 페이지를 제공하는 것.

현재(Phase 11까지)는 개별 세션의 토큰 사용량을 메타 바에서 확인할 수 있지만, 전체적인 사용 패턴과 추이를 파악하려면 파일을 직접 분석해야 한다. Phase 12에서는 Claude Code가 로컬에 저장하는 다양한 데이터 파일을 파싱하여, 차트와 함께 사용량 통계를 한눈에 볼 수 있는 전용 페이지를 제공한다.

## 완료 조건

통계 페이지에서 토큰 사용량, 활동 패턴, 프로젝트별 분석을 차트와 함께 확인 가능.

---

## 현재 상태 (Phase 11 완료)

### 이미 구현된 것

- Claude Code Panel: 타임라인 + 입력창 + 축소 터미널
- 세션 메타 바: 개별 세션의 토큰 사용량 실시간 표시
- 세션 파일 파싱 인프라 (JSONL, fs.watch)
- 프로젝트 디렉토리 ↔ Claude 경로 매핑

### Phase 12에서 추가할 것

- 전용 통계 페이지 (새로운 라우트)
- Claude Code 데이터 파일 파싱 (stats-cache, JSONL, facets, history)
- 개요 / 토큰 / 활동 패턴 / 프로젝트별 / 세션 분석 섹션
- 차트 시각화 (recharts)
- 기간 필터

---

## 데이터 소스

Claude Code가 로컬에 저장하는 파일들을 활용한다.

| 파일 | 경로 | 내용 |
|---|---|---|
| stats-cache | `~/.claude/stats-cache.json` | 일별 활동량, 모델별 토큰, 시간대별 분포 등 집계 데이터 |
| 세션 JSONL | `~/.claude/projects/*/*.jsonl` | 세션별 메시지 원본 (토큰 usage, 모델, 타임스탬프) |
| facets | `~/.claude/usage-data/facets/*.json` | 세션별 목표 달성도, 카테고리, 만족도 분석 |
| history | `~/.claude/history.jsonl` | 전체 사용자 입력 히스토리 |

## 요구사항

### REQ-1: 통계 페이지 진입

사이드바 또는 상단 네비게이션에서 통계 페이지에 진입할 수 있다.

- 사이드바에 통계 아이콘/메뉴 추가
- 클릭 시 전용 통계 페이지로 전환 (기존 터미널 레이아웃과 별도)
- 뒤로가기로 이전 Workspace 레이아웃 복귀

### REQ-2: 개요 섹션

전체 사용량을 요약하는 개요 카드를 표시한다.

- 총 세션 수
- 총 메시지 수
- 오늘 / 이번 주 / 이번 달 사용량
- 첫 사용일부터의 누적 추이 (라인 차트)

### REQ-3: 토큰 사용량 섹션

토큰 사용량을 모델별, 일별로 분석한다.

- 모델별 토큰 사용량 (입력/출력/캐시) — 바 차트 또는 도넛 차트
- 일별 토큰 추이 차트 — 라인 또는 에어리어 차트
- 모델별 비율 (Opus / Sonnet / Haiku) — 도넛 차트

### REQ-4: 활동 패턴 섹션

사용 시간과 빈도를 시각화한다.

- 시간대별 사용 히트맵 (GitHub 잔디 스타일)
- 일별 메시지 수 / 세션 수 / 도구 호출 수 차트
- 요일별 사용 패턴

### REQ-5: 프로젝트별 분석 섹션

프로젝트(Workspace) 단위로 사용량을 비교한다.

- Workspace별 세션 수 / 메시지 수
- 프로젝트별 토큰 소비 비교 — 바 차트

### REQ-6: 세션 분석 섹션

세션 단위의 통계를 분석한다.

- 평균 세션 길이 / 최장 세션
- 세션 카테고리 분포 (코드 작성, 리뷰, 문서 등) — facets 데이터 활용
- 목표 달성도 통계 — facets 데이터 활용

### REQ-7: 기간 필터

통계 데이터의 조회 기간을 선택할 수 있다.

- 필터 옵션: 오늘 / 7일 / 30일 / 전체
- 필터 변경 시 모든 차트 및 지표가 해당 기간에 맞게 갱신
- 기본값: 30일

### REQ-8: 데이터 로딩 전략

빠른 초기 렌더링을 위한 2단계 로딩.

- 1단계: `stats-cache.json` 로드 → 빠른 초기 렌더링 (집계 데이터)
- 2단계: JSONL 파일 파싱 → 상세 분석 (백그라운드 처리)
- 로딩 중 skeleton UI 표시
- 파싱 진행률 표시 (대용량 데이터일 경우)

## 비기능 요구사항

### NFR-1: 초기 로딩 성능

stats-cache.json 기반 개요 지표는 1초 이내에 표시되어야 한다.

### NFR-2: 대용량 데이터 처리

JSONL 파일이 수백 MB일 수 있으므로, 스트리밍 파싱 + 백그라운드 처리로 UI 블로킹을 방지해야 한다.

### NFR-3: 기존 기능 영향 없음

통계 페이지 추가가 터미널, Claude Code Panel 등 기존 기능에 영향을 주지 않아야 한다.

### NFR-4: 차트 렌더링 성능

데이터 포인트가 365개(일별 1년) 이상이어도 차트 렌더링이 부드러워야 한다.

## 범위 제외 (Phase 12에서 하지 않는 것)

| 항목 | 사유 |
|---|---|
| 비용 계산 (달러 환산) | 모델 가격 변동으로 정확도 보장 불가 |
| 실시간 통계 갱신 (WebSocket) | 통계 페이지는 진입 시 로드, 실시간 갱신 불필요 |
| 통계 데이터 내보내기 (CSV, PDF) | 추후 필요 시 구현 |
| 사용량 예산/한도 설정 | 범위 과대 |
| 팀/조직 단위 통계 | 현재 로컬 전용 도구이므로 불필요 |

## 기술 구성

### 데이터 로딩 흐름

```
통계 페이지 진입
├── 1단계: stats-cache.json 로드 (즉시)
│   ├── 개요 카드 렌더링
│   ├── 일별 토큰 추이 차트
│   └── 시간대별 히트맵
├── 2단계: JSONL 파싱 (백그라운드)
│   ├── 프로젝트별 분석
│   ├── 세션 상세 분석
│   └── 모델별 토큰 분리
└── 3단계: facets 로드 (백그라운드)
    ├── 세션 카테고리 분포
    └── 목표 달성도
```

### 차트 라이브러리

- recharts 사용 (기존 프로젝트에 호환)
- **차트 색상은 `--chart-1` ~ `--chart-9` CSS 변수를 사용** (`docs/STYLE.md` 참조)
  - `--chart-1` (ui-blue), `--chart-2` (ui-teal), `--chart-3` (ui-coral) 등
  - 색은 의미를 인코딩: 데이터 의미에 맞는 차트 변수를 선택 (순서대로 사용하지 않음)
  - 예: 입력 토큰 → `var(--chart-1)`, 출력 토큰 → `var(--chart-2)`, 캐시 토큰 → `var(--chart-4)`
- SVG 기반 차트에서 `fill="var(--chart-1)"`, `stroke="var(--chart-5)"` 형태로 적용

## 검증 시나리오

1. **페이지 진입**: 사이드바에서 통계 아이콘 클릭 → 통계 페이지 표시
2. **개요 표시**: 총 세션, 메시지 수, 기간별 사용량이 카드 형태로 표시
3. **토큰 차트**: 일별 토큰 추이가 라인 차트로 표시, 모델별 비율이 도넛 차트로 표시
4. **히트맵**: 시간대별 사용량이 GitHub 잔디 스타일로 표시
5. **기간 필터**: "7일" 선택 → 모든 차트/지표가 최근 7일 데이터로 갱신
6. **프로젝트별**: 각 Workspace의 토큰 소비량이 바 차트로 비교
7. **세션 분석**: 평균/최장 세션 길이, 카테고리 분포가 표시
8. **빠른 로딩**: stats-cache 기반 지표가 1초 이내 표시, 상세 분석은 점진적 로드
9. **대용량 데이터**: 1년치 데이터에서도 차트 렌더링 지연 없음
10. **복귀**: 통계 페이지에서 사이드바 Workspace 클릭 → 이전 터미널 레이아웃 복귀
