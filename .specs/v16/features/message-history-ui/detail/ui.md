# 화면 구성

## 입력창 레이아웃 변경

### 기존

```
┌─────────────────────────────────────────────┐
│  ┌─ 입력창 ──────────────────────────── ⏎ ┐ │
│  │ 메시지를 입력하세요...                  │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 변경

```
┌─────────────────────────────────────────────┐
│  ┌──┐ ┌─ 입력창 ────────────────────── ⏎ ┐ │
│  │🕐│ │ 메시지를 입력하세요...            │ │
│  └──┘ └───────────────────────────────────┘ │
└─────────────────────────────────────────────┘
  ↑                                        ↑
  Clock 아이콘                          Send 버튼
```

- Clock 아이콘: textarea 왼쪽, 기존 flex 컨테이너 안에 추가
- 기존 `flex items-end gap-2` 레이아웃에 Clock 버튼을 첫 번째 자식으로 삽입

## 시계 아이콘 버튼

### 활성 상태

| 속성 | 값 |
|------|-----|
| 아이콘 | lucide-react `Clock` (16px) |
| 컴포넌트 | `Button variant="ghost" size="sm"` |
| 색상 | `text-muted-foreground` |
| hover | `hover:text-foreground hover:bg-muted` |
| padding | 기존 Send 버튼과 동일 (`h-8 w-8 p-0`) |
| 위치 | textarea 왼쪽, `self-end` (하단 정렬) |
| aria | `aria-label="메시지 히스토리"` |

### 비활성 상태

| 조건 | 스타일 |
|------|--------|
| 히스토리 비어있음 | `opacity-50`, `pointer-events-none` |
| `mode === 'disabled'` | `opacity-50`, `pointer-events-none` |

## Popover (데스크톱)

```
┌─ 입력창 영역 ──────────────────────────────┐
│                                            │
│  ┌─ Popover ─────────────────────────────┐ │
│  │  🔍 히스토리 검색...                   │ │
│  │  ─────────────────────────────────────│ │
│  │  이 함수를 리팩토링해줘     3분 전  ✕  │ │
│  │  테스트 코드 작성해줘       1시간 전 ✕ │ │
│  │  커밋해줘                   2시간 전 ✕ │ │
│  │  버그 수정해줘              어제     ✕  │ │
│  │  코드 리뷰해줘              3일 전  ✕  │ │
│  │                                       │ │
│  └───────────────────────────────────────┘ │
│                                            │
│  [🕐] [메시지를 입력하세요...          ] [⏎]│
└────────────────────────────────────────────┘
```

### Popover 컨테이너

| 속성 | 값 |
|------|-----|
| 컴포넌트 | shadcn `Popover` |
| 트리거 | Clock 버튼 |
| 위치 | 버튼 위쪽 (`side="top"`, `align="start"`) |
| 너비 | `w-80` (320px) |
| 최대 높이 | `max-h-[360px]` |
| 내부 | shadcn `Command` 컴포넌트 |

### Command 내부 구조

```
Command
├── CommandInput (검색)
├── CommandList (스크롤 영역, max-h-[300px])
│   ├── CommandEmpty ("히스토리가 없습니다" 또는 "검색 결과가 없습니다")
│   └── CommandGroup
│       └── CommandItem[] (각 항목)
└── (CommandSeparator 불필요)
```

### 항목 (CommandItem) 구성

```
┌─────────────────────────────────────────┐
│  이 함수를 리팩토링해줘       3분 전  ✕ │
│  ↑ 메시지 (truncate)      ↑ 시간  ↑ 삭제│
└─────────────────────────────────────────┘
```

| 요소 | 스타일 |
|------|--------|
| 메시지 텍스트 | `text-sm truncate flex-1 min-w-0` |
| 상대 시간 | `text-xs text-muted-foreground whitespace-nowrap ml-2` |
| 삭제 버튼 | lucide `X` (12px), `opacity-0 group-hover:opacity-100`, `h-5 w-5 p-0` |

- 멀티라인 메시지: 첫 줄만 표시 (`message.split('\n')[0]`)
- 상대 시간: `dayjs(sentAt).fromNow()` (dayjs relative time 플러그인)
- 삭제 버튼: hover 시만 표시, `e.stopPropagation()`으로 이벤트 전파 차단

### 검색

| 속성 | 값 |
|------|-----|
| placeholder | `"히스토리 검색..."` |
| 필터링 | Command 기본 검색 또는 커스텀 filter |
| 동작 | 실시간 필터링 (debounce 없음) |

### 빈 상태

```
┌─ Popover ─────────────────────┐
│  🔍 히스토리 검색...           │
│  ────────────────────────────│ │
│                               │
│    히스토리가 없습니다          │
│                               │
└───────────────────────────────┘
```

- `CommandEmpty` 컴포넌트 활용
- 검색 결과 없을 때: "검색 결과가 없습니다"

### 로딩 상태

```
┌─ Popover ─────────────────────┐
│  🔍 히스토리 검색...           │
│  ────────────────────────────│ │
│                               │
│         (스피너)               │
│                               │
└───────────────────────────────┘
```

- Popover 열릴 때 fetch → 로딩 중 `CommandLoading` 또는 중앙 스피너
- 짧은 로딩이므로 스켈레톤 대신 스피너 사용

### 에러 상태

```
┌─ Popover ─────────────────────┐
│  🔍 히스토리 검색...           │
│  ────────────────────────────│ │
│                               │
│    불러오기 실패               │
│    [다시 시도]                 │
│                               │
└───────────────────────────────┘
```

- "다시 시도" 클릭 시 재 fetch

## Drawer (모바일)

```
┌─────────────────────────────┐
│                             │
│  [터미널/타임라인 영역]       │
│                             │
├─────────────────────────────┤ ← Drawer 핸들
│  🔍 히스토리 검색...         │
│  ───────────────────────── │
│  이 함수를 리팩토링해줘  3분 전  ✕ │
│  테스트 코드 작성해줘   1시간 전  ✕ │
│  커밋해줘             2시간 전  ✕ │
│  버그 수정해줘         어제     ✕ │
│  코드 리뷰해줘        3일 전   ✕ │
└─────────────────────────────┘
```

- `useIsMobileDevice()` 훅으로 데스크톱/모바일 분기
- 모바일: shadcn `Drawer` (하단에서 올라옴) + 내부 `Command`
- 항목 구성, 검색, 빈/로딩/에러 상태: 데스크톱과 동일
- Drawer 높이: `snap-points` 사용 또는 `max-h-[60vh]`
- 터치 타겟: 항목 높이 최소 44px

## 반응형/접근성

| 항목 | 처리 |
|------|------|
| Clock 버튼 | `tabIndex={0}`, Enter/Space로 Popover 토글 |
| Command 항목 | 화살표 키로 탐색, Enter로 선택 |
| 삭제 버튼 | Tab으로 접근, Enter/Space로 삭제 |
| Popover 열림 | 검색 입력에 자동 포커스 |
| Popover 닫힘 | textarea로 포커스 복귀 |
| Escape | Popover/Drawer 닫기 |

## 컴포넌트 설치

Command, Drawer 컴포넌트가 아직 없으므로 설치 필요:

```bash
npx shadcn@latest add command
npx shadcn@latest add drawer
```
